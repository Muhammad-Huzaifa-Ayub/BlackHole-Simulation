# Measured Performance

Benchmark performed at **800 × 600** resolution.

| Metric | Result |
|--------|---------|
| Average FPS | **32–33 FPS** |
| Average Frame Time | **≈30.5 ms** |
| Pixels Processed | **15.45 Million Pixels/sec** |
| RK4 Geodesic Integrations | **3.86 Billion Steps/sec** |
| Estimated Compute Throughput | **≈107 GFLOPS** |

These measurements were obtained using the built-in benchmark mode after the warm-up phase.

---

# Display

Supported window resolutions include:

- 800 × 600
- 1280 × 720
- 1600 × 900
- 1920 × 1080

Higher resolutions increase the number of pixels integrated per frame and therefore reduce frame rate proportionally.

---

# Performance Scaling

The renderer performs one independent relativistic ray integration per pixel using a fixed **250-step fourth-order Runge–Kutta (RK4)** integrator.

Consequently,

- increasing screen resolution increases computational cost almost linearly,
- reducing resolution increases frame rate nearly proportionally,
- camera position has only a minor effect on performance because every pixel follows the same integration budget.

---

# Notes

- The renderer executes entirely on the GPU using OpenGL 4.3 fragment shaders.
- Vertical synchronization (V-Sync) is disabled by default for unrestricted benchmarking.
- Performance depends primarily on GPU compute capability and pixel throughput.
- The benchmark statistics generated by the application correspond to the measured rendering workload during the benchmark interval.
